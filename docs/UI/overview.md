# UI/UX 设计概览

> 忆流 V2.0 界面设计和用户体验原则

---

## 一、设计原则

### 1.1 核心理念

| 原则 | 说明 | 示例 |
|------|------|------|
| **简洁** | 减少干扰，专注内容 | 隐藏不必要的菜单 |
| **快速** | 3 秒内完成操作 | 默认聚焦输入框 |
| **一致** | 交互模式统一 | 相同的反馈方式 |
| **可预测** | 结果明确可见 | 保存成功提示 |

### 1.2 设计参考

参考 CherryStudio 的设计风格:

- 简洁的侧边栏
- 清晰的内容区域
- 自然的间距
- 一致的图标风格

## 四、交互设计

### 4.1 输入区域

**位置**: 页面底部固定

**规格**:

| 属性 | 值 |
|------|-----|
| 高度 | 80px (可扩展) |
| 内边距 | 16px |
| 圆角 | 12px |
| 背景 | --color-bg-secondary |
| 边框 | 1px solid --color-border |

**状态**:

| 状态 | 样式 |
|------|------|
| 默认 | 浅色背景 |
| 聚焦 | 主色边框 |

**位置**: 输入框右侧

**规格**:

| 属性 | 值 |
|------|-----|
| 宽度 | 80px |
| 高度 | 36px |
| 背景 | --color-primary |
| 文字 | 白色 |
| 圆角 | 8px |

**状态**:

| 状态 | 样式 |
|------|------|
| 默认 | 主色背景 |
| 悬停 | 深主色 |
| 禁用 | 50% 透明度 |
| 加载 | 显示 spinner |

### 4.3 笔记卡片

**规格**:

| 属性 | 值 |

| 圆角 | 12px |

## 六、反馈设计

### 6.1 加载状态

**骨架屏**: 笔记列表加载时显示

**Spinner**: 按钮加载时显示

### 6.2 成功反馈

**Toast**: 操作成功后显示

```
┌────────────────────────────────┐
│ ✅ 笔记已保存                   │
└────────────────────────────────┘
```

### 6.3 错误反馈

**Toast**: 操作失败后显示

```
┌────────────────────────────────┐
│ ❌ 保存失败，请重试             │
└────────────────────────────────┘
```

### 6.4 空状态

**无笔记**:

```
┌────────────────────────────────┐
│                                │
│      📝                       │
│                                │
│   还没有笔记，来记一条吧！      │
│                                │
└────────────────────────────────┘
```

## 七、响应式设计

### 7.1 断点

| 断点 | 宽度 | 布局变化 |
|------|------|----------|
| mobile | < 768px | 单列布局 |
| tablet | 768px - 1024px | 窄侧边栏 |
| desktop | > 1024px | 完整布局 |

### 7.2 移动端适配

| 元素 | 适配 |
|------|------|
| 侧边栏 | 折叠为底部导航 |
| 输入框 | 全宽 |
| 卡片 | 全宽，减少内边距 |

## 八、可访问性

### 8.1 键盘导航

| 操作 | 快捷键 |
|------|--------|
| 新建笔记 | Cmd/Ctrl + N |
| 搜索 | Cmd/Ctrl + K |
| 设置 | Cmd/Ctrl + , |
| 关闭 | Escape |

### 8.2 ARIA 支持

- 所有交互元素有 aria-label
- 图标按钮有 aria-hidden
- 模态框有 role="dialog"
- 表单元素关联标签
