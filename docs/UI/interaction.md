## 三、各页面交互

### 3.1 即时灵感

**入口**: 左侧导航「即时灵感」
**右侧面板**: ✅ 显示（记录后可即时查看保存结果）

**功能**:
- 对话式输入框
- 发送消息（Enter 键或点击发送按钮）
- AI 自动回复并添加标签
- 消息历史展示
- 快捷键 ⌘K 快速聚焦
- **双击编辑用户消息 + 重新生成标签**

**双击编辑交互**:
| 操作 | 结果 |
|------|------|
| 双击用户消息 | 进入编辑模式（显示输入框+当前标签） |
| 修改内容 | 点击保存时重新生成标签 |
| 点击「保存」 | 更新内容 + 重新生成标签 + 更新右侧列表 |
| 点击「取消」 | 恢复原内容，退出编辑模式 |

**交互流程**:
```
用户输入 → 发送 → 显示用户消息 → AI 处理 → 显示回复 → 保存到笔记列表 → 右侧列表自动刷新
                              ↓
                    双击编辑（可选） → 重新保存 + 重新生成标签
```

**交互元素**:
| 元素 | 行为 |
|------|------|
| 输入框 | 支持多行，Enter 发送，Shift+Enter 换行 |
| 发送按钮 | 点击发送 |
| 消息气泡 | 展示历史记录，双击进入编辑模式 |
| AI 回复 | 自动生成标签，只读 |
|------|------|
| 输入框 | 支持多行，Enter 发送，Shift+Enter 换行 |
| 发送按钮 | 点击发送 |
| 消息气泡 | 展示历史记录 |
| AI 回复 | 自动生成标签 |

### 3.2 写笔记

**入口**: 左侧导航「写笔记」
**右侧面板**: ❌ 隐藏（专注编辑场景）

**功能**:
- 富文本编辑器（新建或续写）
- 工具栏（加粗、斜体、标题、列表、链接）
- 手动输入 `#标签`
- AI 自动推荐标签（混合模式）
- 字数统计
- 保存笔记

**场景区分**:
| 模式 | 触发方式 | 顶部状态栏 |
|------|----------|-----------|
| 新建笔记 | 进入「写笔记」默认显示空白编辑器 | 显示「🆕 新建笔记」 |
| 续写旧笔记 | 从右侧笔记列表点击「编辑」进入 | 显示「✏️ 续写：[笔记标题]」 |

**续写行为（简化版）**:
- 续写模式保存时：**默认另存为新笔记**，不覆盖原笔记
- 原笔记保留，新笔记创建

**标签混合模式**:
| 类型 | 说明 | 视觉 |
|------|------|------|
| 手动标签 | 用户主动输入 `#主题` | 深色背景 (`#2D3748`)，不可点击删除 |
| AI推荐标签 | AI 事后推荐补充 | 浅色背景 (`#E2E8F0`)，右侧带 × 可关闭 |

**AI 标签推荐时机**: 保存时或编辑后 3 秒无操作

**交互流程**:
```
写笔记入口
    ↓
┌─────────────────────────────────────────┐
│  场景 A: 新建                            │
│  - 默认空白编辑器                        │
│  - 输入内容                             │
│  - 保存                                 │
├─────────────────────────────────────────┤
│  场景 B: 续写                           │
│  - 从右侧列表点击打开                   │
│  - 编辑已有内容                        │
│  - 保存                                 │
└─────────────────────────────────────────┘
    ↓
输入 #标签 → 自动识别保存
    ↓
保存 → 添加到笔记列表
```

**交互元素**:
| 元素 | 行为 |
|------|------|
| 工具栏按钮 | 点击应用格式 |
| 编辑区域 | contenteditable，支持富文本 |
| #标签输入 | 检测 `#` 开头自动识别为标签 |
| 字数统计 | 实时更新 |
| 保存按钮 | 点击保存到笔记 |

### 3.3 知识库

**入口**: 左侧导航「知识库」
**右侧面板**: ✅ 显示

**功能**（合并后）:
- **全部笔记列表**：展示所有向量化存储的笔记
- **标签筛选**：点击标签过滤笔记
- **分类卡片**：聚合视图，点击查看该分类下的笔记
- **全能力搜索**：顶部搜索框，支持关键词 + 语义搜索

**搜索说明**:
- 所有笔记自动向量存储
- 搜索时同时支持精确匹配和语义匹配
- 搜索结果按相关度排序

**交互流程**:
```
进入知识库
    ↓
┌─────────────────────────────────────┐
│ 默认显示：全部笔记列表              │
│ 用户操作：                          │
│   - 点击标签 → 按标签筛选           │
│   - 点击分类卡片 → 按分类筛选       │
│   - 输入搜索 → 全能力搜索          │
└─────────────────────────────────────┘
    ↓
点击笔记 → 查看详情
```

**交互元素**:
| 元素 | 行为 |
|------|------|
| 搜索框 | 输入关键词，实时搜索 |
| 标签按钮 | 点击筛选该标签下的笔记 |
| 分类卡片 | 点击查看该分类的笔记列表 |
| 笔记列表项 | 点击查看详情 |

### 3.4 导出备份

**入口**: 左侧导航「导出备份」
**右侧面板**: ❌ 隐藏（与笔记列表无关）

**功能**:

- JSON 导出

**交互流程**:

```
选择导出格式 → 点击导出 → 下载文件
```

### 3.4 笔记列表/详情（右侧面板）

**入口**: 右侧固定面板（根据当前入口显隐）

**显示逻辑**:
| 入口 | 显示 | 默认视图 |
|------|------|----------|
| 即时灵感 | ✅ | 列表 |
| 写笔记 | ❌ | - |
| 知识库 | ✅ | 列表 |
| 导出备份 | ❌ | - |
| 设置 | ❌ | - |

**功能**:
- 列表视图：展示所有笔记
- 详情视图：展示单条笔记内容
- 切换视图
- 编辑/删除笔记

**交互元素**:
| 元素 | 行为 |
|------|------|
| 笔记卡片 | 点击查看详情 |
| 列表/详情切换 | 点击切换视图 |
| 编辑按钮 | 跳转到写笔记页面，加载内容（进入续写模式） |
| 删除按钮 | 确认后删除笔记 |

## 五、右侧面板显隐实现

### 5.1 显示规则

```typescript
interface RightPanelConfig {
  [key: string]: {
    show: boolean;      // 是否显示
    defaultView: 'list' | 'detail';  // 默认视图
  }
}

const rightPanelConfig: RightPanelConfig = {
  '/': { show: true, defaultView: 'list' },           // 即时灵感
  '/write': { show: false, defaultView: 'list' },      // 写笔记
  '/knowledge': { show: true, defaultView: 'list' },   // 知识库
  '/search': { show: true, defaultView: 'list' },      // 语义搜索
  '/export': { show: false, defaultView: 'list' },     // 导出备份
};
```

### 5.2 状态切换

```
用户点击左侧导航项
    ↓
根据当前路由查表获取配置
    ↓
右侧面板执行 show/hide + 动画过渡（200ms）
    ↓
如显示，设置默认视图
```

## 六、数据流

```
用户输入
    ↓
前端处理 (输入验证)
    ↓
业务逻辑 (创建/搜索/删除)
    ↓
数据存储 (LibSQL)
    ↓
向量处理 (@xenova/transformers)
    ↓
UI 更新
```

## 七、状态管理

| 状态 | 说明 |
|------|------|
| notes | 笔记列表数据 |

| currentPanel | 当前显示的面板 |

## 八、边界情况

| 场景 | 处理 |
|------|------|
| 空输入 | 禁用发送按钮 |
| 网络错误 | 显示错误提示 |
| 搜索无结果 | 显示空状态 |
| 删除确认 | 二次确认对话框 |
| 续写保存 | 自动另存为新笔记 |
