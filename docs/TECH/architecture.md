## 三、数据流

### 3.1 记录流程

```
用户输入 → 前端组件 → IPC 通信 → 主进程 → 业务逻辑 → 数据库
     ↓
  响应返回 → IPC 通信 → 前端更新 → UI 反馈
```

### 3.2 搜索流程

```
用户输入 → 前端组件 → IPC 通信 → 主进程 → 搜索服务 → 向量匹配
     ↓
  结果返回 → IPC 通信 → 前端渲染 → 展示结果
```

### 3.3 向量化流程

```
新笔记创建 → 检查 AI 能力 → 生成向量 → 存储向量 → 更新索引
     ↓
  失败处理 → 记录日志 → 跳过向量化 → 继续流程
```

## 四、IPC 通信

### 4.1 通道定义

| 通道 | 方向 | 用途 |
|------|------|------|
| `note:create` | renderer → main | 创建笔记 |
| `note:list` | renderer → main | 获取笔记列表 |
| `note:get` | renderer → main | 获取单条笔记 |
| `note:update` | renderer → main | 更新笔记 |
| `note:delete` | renderer → main | 删除笔记 |
| `search:semantic` | renderer → main | 语义搜索 |
| `search:keyword` | renderer → main | 关键词搜索 |
| `stats:get` | renderer → main | 获取统计 |
| `export:markdown` | renderer → main | 导出 Markdown |

### 4.2 通信格式

```typescript
// 请求格式
interface IpcRequest<T = any> {
  channel: string;
  data: T;
}

// 响应格式
interface IpcResponse<T = any> {
  success: boolean;
  data?: T;
  error?: string;
}
```

## 五、关键决策

### 5.1 为什么选择 Electron?

| 因素 | 理由 |
|------|------|
| 跨平台 | 一套代码，Mac/Windows/Linux |
| 生态 | 成熟稳定，文档丰富 |
| 调试 | Chrome DevTools 易用 |
| 包体积 | 可接受（相比 Tauri 稍大） |

### 5.2 为什么选择 LibSQL?

| 因素 | 理由 |
|------|------|
| 向量支持 | 原生向量字段 |
| 纯 JS | 无需额外依赖 |
| SQLite 兼容 | 迁移成本低 |
| 性能 | 写入速度快 |

### 5.3 为什么选择本地向量?

| 因素 | 理由 |
|------|------|
| 隐私 | 数据不外传 |
| 离线 | 无网也能用 |
| 成本 | 无 API 费用 |
| 响应 | 无网络延迟 |

## 六、安全设计

### 6.1 数据安全

- 所有数据本地存储
- 可选 WebDAV 加密同步
- 不收集用户数据

### 6.2 网络安全

- AI 功能可选（需用户配置 API Key）
- 不强制要求网络
- 敏感操作需确认

### 6.3 权限管理

- 文件系统: 仅访问数据目录
- 网络: 可选（AI 功能）
- 通知: 可选
