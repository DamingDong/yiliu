# 产品概览

> 忆流 - 陪伴用户记录，沉淀知识的第二大脑

---

## 一、产品定位

| 维度 | 内容 |
|------|------|
| **产品名称** | 忆流 (Yiliu) |
| **产品类型** | 桌面端 AI 知识管理应用 |
| **价值主张** | **第二大脑** - 陪伴用户记录，沉淀知识 |
| **核心理念** | 像使用笔记本一样简单，像第二大脑一样智能 |

---

## 二、目标用户

### 主要用户

| 用户群体 | 场景 | 痛点 |
|----------|------|------|
| **职场人士** | 会议记录、项目笔记、工作心得 | 笔记分散、难以查找 |
| **个人用户** | 学习笔记、读书笔记、生活记录 | 缺乏整理、碎片化 |
| **学习人群** | 学术笔记、研究资料、知识沉淀 | 信息过载、难以复用 |

### 用户画像

```
典型用户: 小明
- 年龄: 28 岁
- 职业: 产品经理
- 日均笔记: 5-10 条
- 主要场景: 会议记录、工作思考、学习笔记
- 痛点: 
  - 微信记录太分散
  - 笔记软件太复杂
  - 想要时找不到
- 期望: 
  - 快速记录（< 3 秒）
  - 自动整理
  - 智能搜索
```

## 三、产品阶段

### Phase 1: 记录入口 (MVP)

**目标**: 像用笔记本一样记录

| 功能 | 描述 |
|------|------|
| 快速捕捉 | 对话式输入（语音/文字），3 秒内完成记录 |
| 笔记列表 | 时间线展示，类似微信对话 |
| 本地存储 | 数据本地保存，安全可控 |

**用户旅程**:
```
打开应用 → 输入想法 → 自动保存 → 退出
```

### Phase 2: 向量化能力

**目标**: 记录的东西能被找到

| 功能 | 描述 |
|------|------|
| 向量存储 | 实时向量化，存储嵌入向量 |
| 语义搜索 | 理解意图，找你想找 |
| 混合搜索 | 语义 + 关键词双重检索 |

**用户旅程**:
```
打开应用 → 搜索"项目进展" → 找到相关笔记
```

### Phase 3: 自动整理

**目标**: 记录的东西自动组织

| 功能 | 描述 |
|------|------|
| 自动标签 | AI 自动生成标签 |
| 自动摘要 | 自动生成笔记摘要 |
| 智能分类 | 按主题/时间自动归类 |

**用户旅程**:
```
记录 → 自动添加标签 → 后续可按标签筛选
```

### Phase 4: AI 增强

**目标**: AI 辅助创作和扩展

| 功能 | 描述 |
|------|------|
| AI 扩写 | 基于笔记内容扩展写作 |
| AI 改写 | 优化表达方式 |
| 互联网整合 | 结合外部信息扩展 |

**用户旅程**:
```
选择笔记 → AI 扩写 → 导出使用
```

## 四、竞品对比

| 产品 | 定位 | 优点 | 不足 |
|------|------|------|------|
| **Notion** | All-in-one 工作空间 | 功能全面、生态丰富 | 上手复杂、价格高 |
| **Obsidian** | 双链笔记 | 高度可定制、本地优先 | 技术门槛高 |
| **印象笔记** | 老牌笔记 | 跨平台、稳定 | AI 功能弱 |
| **CherryStudio** | AI 客户端 | 多模型、知识库 | 非专注笔记 |
| **忆流** | 轻量笔记 + AI | 简单、快速、AI 增强 | 功能较少 |

---

## 五、核心差异点

### 1. 极简入口
- 类似微信对话的输入体验
- 无需学习成本，打开即用

### 2. 轻量记录
- 不需要分类、标签、层级
- 记录即整理，AI 自动处理

### 3. 桌面端优先
- Mac/PC 原生应用
- 系统级集成（托盘、快捷键）

### 4. 本地优先
- 数据完全本地存储
- 可选云同步（WebDAV）

## 六、成功指标

| 指标 | 目标 (V2.0) |
|------|-------------|
| 记录转化率 | > 80% (输入 → 保存) |
| 搜索成功率 | > 70% (搜索 → 找到) |
| 平均记录时间 | < 3 秒 |
| 启动时间 | < 2 秒 |

---

## 七、限制与边界

### 不做的事情

- 不做复杂的文档排版
- 不做多人协作
- 不做移动端（初期）
- 不做复杂的知识图谱

### 技术限制

- 纯本地运行，不支持实时同步
- 嵌入向量依赖本地模型（无网络时降级）
- 单用户场景
