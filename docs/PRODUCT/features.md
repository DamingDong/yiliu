## 二、P0 核心功能

### 2.1 快速记录

#### 功能描述

用户可以通过对话式输入快速记录想法，无需复杂操作。

#### 用户交互

```
┌─────────────────────────────────────────────┐
│  📝 记录                                    │
├─────────────────────────────────────────────┤
│                                             │
│  ┌─────────────────────────────────────┐    │
│  │ 输入你的想法...                       │    │
│  └─────────────────────────────────────┘    │
│                                             │
│           [ 发送 ]                          │
│                                             │
└─────────────────────────────────────────────┘
```

#### 输入方式

| 方式 | 说明 | 示例 |
|------|------|------|
| 文字输入 | 直接输入文字 | "今天开会讨论了项目进展" |
| 回车发送 | 回车键提交 | - |
| 点击发送 | 点击按钮提交 | - |

#### 行为规则

1. 输入非空时启用发送按钮
2. 发送后清空输入框
3. 自动保存到数据库
4. 实时返回保存结果
5. 记录时间戳（创建时间）

#### 技术实现

```typescript
interface NoteInput {
  content: string;      // 笔记内容
  source?: string;      // 来源: 'text' | 'voice' | 'link'
  url?: string;         // 如果是链接，存储 URL
}

interface NoteOutput {
  id: string;
  content: string;
  createdAt: number;
  updatedAt: number;
}
```

### 2.2 写笔记（长文编辑）

#### 功能描述

手动输入长篇笔记，支持富文本编辑和主题标签。

#### 两种场景

| 场景 | 触发方式 | 说明 |
|------|----------|------|
| 新建笔记 | 进入「写笔记」默认显示 | 空白编辑器，从零开始写 |
| 续写旧笔记 | 从右侧笔记列表点击打开 | 加载已有内容继续编辑 |

#### 标签系统（混合模式）

| 类型 | 说明 | 示例 |
|------|------|------|
| 手动标签 | 用户主动输入 `#主题` | 输入 `#技术 #学习` |
| 自动标签 | AI 事后推荐补充 | AI 建议添加 `#项目` |

#### 标签检测规则

1. 用户输入时实时检测 `#` 开头的内容
2. `#` 后面的连续字符识别为主题
3. 多个 `#主题` 用空格分隔
4. AI 会在保存后推荐补充标签

#### 用户交互

```
┌─────────────────────────────────────────────┐
│  ✏️ 写笔记                                  │
├─────────────────────────────────────────────┤
│  ┌─────────────────────────────────────┐    │
│  │ [B] [I] [H] [≡] [🔗]                │    │
│  ├─────────────────────────────────────┤    │
│  │                                     │    │
│  │ 会议记录：                          │    │
│  │ #会议 #项目                         │    │
│  │                                     │    │
│  │ 今天参加了项目周会，               │    │
│  │ 讨论了以下内容...                  │    │
│  │                                     │    │
│  └─────────────────────────────────────┘    │
│                                             │
│  86 字                        [保存笔记]    │
└─────────────────────────────────────────────┘
```

#### 交互元素

| 元素 | 行为 |
|------|------|
| 工具栏 | 加粗、斜体、标题、列表、链接 |
| 编辑区 | 富文本输入，支持拖拽 |
| #标签 | 输入 `#` 自动识别主题 |
| 字数统计 | 实时更新显示 |
| 保存按钮 | 保存到笔记列表 |

### 2.4 向量存储

#### 功能描述

记录时自动生成向量嵌入，存储用于语义搜索。

#### 技术实现

| 组件 | 说明 |
|------|------|
| 嵌入模型 | @xenova/transformers (all-MiniLM-L6-v2) |
| 向量维度 | 384 维 |
| 存储方式 | LibSQL BLOB 字段 |
| 向量化时机 | 记录时实时向量化 |

#### 数据流程

```
用户输入 → 内容存储 → 向量化 → 向量存储 → 返回结果
```

#### 降级策略

| 情况 | 处理方式 |
|------|----------|
| 无网络 + 无本地模型 | 仅存储文本，不向量化 |
| 模型加载失败 | 记录异常日志，跳过向量化 |
| 向量化超时 | 10 秒超时，后台重试 |

### 2.5 知识库（整合搜索）

#### 功能描述

知识库是所有笔记的统一存储和展示入口，包含以下能力：
- **笔记列表**：展示所有向量化存储的笔记
- **标签筛选**：点击标签过滤笔记
- **分类卡片**：聚合视图，点击查看该分类下的笔记
- **全能力搜索**：顶部搜索框，支持关键词 + 语义搜索

#### 导航结构变化

**优化前**：
```
工具
├── 知识库（仅分类卡片）
├── 语义搜索（独立入口）
└── 导出备份
```

**优化后**：
```
工具
├── 知识库（整合浏览 + 筛选 + 搜索）
└── 导出备份
```

#### 用户交互

```
┌─────────────────────────────────────────────────────┐
│ 知识库                          🔍 搜索笔记...      │
├─────────────────────────────────────────────────────┤
│ [全部] [技术] [工作] [读书] [想法]                  │
├─────────────────────────────────────────────────────┤
│ ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐    │
│ │ 技术学习 │ │ 工作项目 │ │ 读书笔记 │ │ 想法创意 │    │
│ │  12篇   │ │   8篇   │ │  15篇   │ │  23篇   │    │
│ └─────────┘ └─────────┘ └─────────┘ └─────────┘    │
├─────────────────────────────────────────────────────┤
│ 笔记列表：                                          │
│ ┌─────────────────────────────────────────────┐    │
│ │ 开会被问到项目的进展...  #会议 #项目  10:30│    │
│ └─────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────┘
```

#### 搜索能力

搜索功能内嵌于知识库页面，支持：
1. **语义搜索**：基于向量相似度搜索，理解用户意图
2. **关键词搜索**：传统关键词匹配，作为语义搜索的补充
3. **自动回退**：语义搜索无结果时，自动回退到关键词搜索

#### 搜索流程

1. 用户输入搜索词
2. 先尝试语义搜索（向量相似度）
3. 如无结果，回退到关键词匹配
4. 按相关度排序显示结果

#### 匹配度计算

```
匹配度 = cosine_similarity(query_vector, note_vector) * 100%
```

## 三、P1 重要功能

### 3.1 关键词搜索

#### 功能描述

传统关键词匹配，作为语义搜索的补充和回退。

#### 匹配规则

- 不区分大小写
- 空格分隔多关键词（AND 逻辑）
- 匹配内容字段

#### 回退场景

1. 语义搜索无结果
2. AI 能力不可用
3. 向量化失败

### 3.3 笔记删除

#### 功能描述

删除单条笔记。

#### 用户交互

- 列表页：滑动/右键删除
- 详情页：删除按钮

#### 确认逻辑

- 普通笔记：直接删除
- 重要笔记：二次确认

- 不物理删除，标记为 deleted
- 可恢复（版本历史）

### 4.2 版本历史

#### 功能描述

记录笔记的历史版本，可追溯和回滚。

#### 版本策略

- 自动保存：每次编辑创建新版本
- 最多保留：50 个版本
- 标记版本：用户可标记重要版本

#### 版本信息

| 字段 | 说明 |
|------|------|
| version | 版本号 (1, 2, 3...) |
| content | 版本内容 |
| createdAt | 创建时间 |
| isMarked | 是否标记 |
| markNote | 标记备注 |
